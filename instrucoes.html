Instruções para fazer CSS para o Be on the Net

Olá, designer. Você acaba de receber um pacote com todas as páginas que existem no Be on the Net até agora. Esta página contém instruções e macetes para entender melhor o sistema Be on the Net e seu HTML.

Para começo de conversa: o que eu, designer, tenho de fazer afinal?

Bem simples. Você tem de pegar os arquivos HTML contidos neste pacote e dar todo um novo layout a ele. Primeiro, através de uma prévia em forma de imagem de Photoshop; depois, através de um CSS só seu.

Estas páginas, por enquanto, estão com o CSS que as tornam semelhantes ao site da fotógrafa de casamentos Patricia Figueira. Imagine que ela contratou você pedindo uma reformulação geral do layout do site. Pois é esta a tarefa: usar o HTML já existente e construir um novo template sobre ele, usando puro CSS.

Como designer, você não precisa tocar no HTML nem no JavaScript — aliás, recomendamos que, para que seu CSS seja compatível com o Be on the Net, você realmente mantenha intactos os HTML’s destes arquivos! Ou melhor: a única parte dos HTML’s em que você vai mexer livremente são as tags link dentro do head, que chamam seus arquivos CSS.

Como vai ficar a aparência, como vão ser diagramados os thumbnails das imagens e os outros elementos, isso só é limitado pela sua criatividade. Envie-nos seu layout de Photoshop e daremos nosso feedback para você começar seu CSS. As instruções a seguir vão servir como orientação para você se encontrar no HTML e saber todos os macetes necessários para levar sua arte aos limites.

O que você vê no site:

Basicamente, o que você viu em comum no site da Patricia e nos arquivos deste pacote foram:

Imagens
A maior parte do site é feita de imagens. Elas são o carro-chefe do Be on the Net. Você vê galerias, vê imagens grandes, brinca com a seleção de imagens. É o maior playground do sistema;
Texto
Na página “Quem somos”, há texto. É basicamente um discurso institucional, para dar uma seduzida no usuário com palavras — se ele já não foi seduzido pelas imagens;
Um formulário de contato
Há uma página para o usuário entrar em contato com a Patricia. Se quiser, pode mandar uma mensagem a ela dizendo que está na equipe de designers do Be on the Net!   :)
Erros
Se você teve azar o bastante (ou sorte, dependendo do ponto de vista!) acessando o site dela, você se deparou com uma das três divertidas páginas de erro: página não encontrada, servidor sentimental, e repositório preguiçoso.

Um sistema bem simples, como você pode ver.

Os módulos do Be on the Net:

Agora dê uma nova olhada no código HTML dos arquivos numerados deste pacote. Mais precisamente na tag body de cada um. Repare que cada um deles tem pelo menos um id e uma classe.

Se você der a esta tag body e a seus id’s e classes carinho, colinho e atenção, já vai dar um grande passo em direção a fazer o sistema se dobrar ao seu design.

A última palavra de cada classe reflete um esquema de divisão que criamos (Ex.: body id="index" class="instructions text"). Be on the Net é um sistema dividido em “módulos”, ou bundles, como temos chamado na Improve It. Hoje, existem cinco módulos ou bundles básicos:

Pictures
É o módulo das imagens, o carro-chefe do Be on the Net. É o que concentra o maior número de páginas e funcionalidades;
Text
O módulo institucional. Compreende páginas textuais — neste caso, a página “Quem somos”;
Contact
O módulo de contato do sistema. É composto por uma única página de contato, com algumas possibilidades de comportamento;
Application
O módulo das páginas de erro. São três: o erro 404 (página não encontrada), 500 (erro de servidor), e timeout (servidor ocupado);
Videos
Módulo dos vídeos. Compreende basicamente duas páginas: a lista de todos os vídeos, e a visualização de um vídeo em tamanho grande;

... Familiar? Agora você pode perceber que o início de cada arquivo deste pacote leva um destes nomes: pictures, text, contact, application, ou [inédito no site da Patricia] videos.

HTML e a flexibilidade do CSS:

Não é possível mexer no HTML nem no JavaScript (mas se você tiver alguma sugestão de mudança ou retoque, não hesite em falar, que a gente vê o que dá para fazer!). Procuramos fazer um HTML limpo, compreensível e rico o bastante para que você, designer, consiga fazer a festa com puro CSS. Em todo o HTML há dicas e ganchos que o designer pode usar a seu favor, tais como:

Classes e id’s na tag body:
Como dissemos, a última palavra da classe de cada página indica seu módulo. Mas as próprias classes e id’s podem formar várias combinações para que você consiga estilizar páginas inteiras, ou seqüências de páginas semelhantes, ou mesmo inserir uma particularidade numa única página! Mais sobre o assunto lá na frente;
Tags variadas
Procuramos enxugar em classes e id’s nas tags, e em compensação elas são super variadas; cada elemento que há dentro de outro é praticamente único, o que dá excelentes oportunidades para brincadeiras como múltiplas imagens de background, entre outras;
Tags salvadoras para o IEca, image replacement e outras firulas
Em alguns pontos do código há certas tags quietinhas lá na delas, esperando uma oportunidade de entrar em ação. São como o <span class="ie"> nas páginas de thumbnails para centralizar imagens verticalmente, os spans vazios para image replacement na página de foto full-size, ou o <li="empty"> no final de cada lista de fotos, para quebrar floats com facilidade. Mais detalhes e macetes lá na frente.

A divisão do CSS:

Dê uma olhada na pasta stylesheets. Lá estão todas as folhas de estilo que compõem este modelo de design, parecido com o da Patricia Figueira. Podemos ver que há uma divisão incomum entre tipografia, layout e cores.

Esta divisão não é obrigatória. Você pode utilizar quantas folhas de estilo preferir, e dividir a informação entre elas como quiser. Esta foi somente a opção que o Leandro fez para estilizar as páginas. Esta organização tem-se mostrado bastante útil em nossos projetos, já que é legível para qualquer desenvolvedor, permite-nos encontrar e modificar seletores com facilidade, e leva em consideração as três mídias mais usadas (screen, print e handheld). Há um artigo dele em nosso blog que explica em detalhes o porquê de separar a tipografia.

Caso você queira experimentar esta divisão, seguem os detalhes:

typography.css
É a folha de estilo que cuida de tudo relacionado a tipografia: tamanho de fonte, maiúsculas, minúsculas, negrito, itálico, espaçamento entre letras, palavras e linhas etc. É aceita sem qualquer bug por todos os browsers, não precisando de qualquer regra de exceção. É também a única folha de estilo que leva o atributo media="all", garantindo um mínimo de identidade visual mesmo quando o meio é limitado em cores (print) ou em imagens (handheld);
layout.css
A folha de estilo que cuida da diagramação dos elementos na página: margens, paddings, posicionamento, disposição em camadas, entre outros. Como a diagramação é o fator que mais depende do meio veiculado e do browser, geralmente precisa de outros arquivos de suporte (nota: Leandro só cuidou do layout de tela; poderia ter feito também arquivos layout_print e layout_handheld — e você também pode ficar à vontade para criar os seus!). Por causa da possibilidade de o IEca não obedecer a certas regras de layout, o layout.css é a única folha de estilos que pode necessitar de exceções para o IE (repare nos arquivos layout_ie.css e layout_ie6.css: neles, só há seletores relativos à diagramação de elementos);
color.css
Cuida dos elementos de cor do site: cor do texto, do fundo, das bordas, e a aplicação de imagens de background. Também é uma folha de estilo bem aceita por todos os browsers, por isso é muito raro dar confusão. Com este arquivo, pode-se manter sob controle toda a paleta de cores do site (repare que no início do arquivo 03_color_screen_handheld.css estão anotadas todas as cores usadas, para aplicação rápida e fácil). Todas as imagens usadas no site vêm da pasta /images neste pacote e, com exceção do logotipo, são aplicadas na folha de estilo de cor.

Conhecendo o body — módulos, páginas e situações:

Até hoje, existem, como mencionados antes, cinco módulos no Be on the Net: pictures, text, contact, application e videos. Agora você vai entender o porquê dos nomes esquisitos e gigantescos de cada arquivo deste pacote. Depois de explicado, você vai saber se guiar direitinho por entre eles.

Vamos destrinchar o nome do arquivo 04(pictures)_best_galleries_selection(04_pictures_in_cart).html (putz!). É assim:

  • O número no início (04) diz a página a que o arquivo se refere. Neste caso, é à quarta página do módulo pictures — que trata da seleção de fotos;
  • O primeiro par de parênteses ((pictures)) apresenta o módulo. O arquivo em questão pertence ao módulo pictures;
  • A seqüência de palavras que há entre o primeiro e o segundo pares de parênteses (_best_galleries_selection) reflete a combinação de classes e id’s que aparece na tag body deste arquivo. Neste exemplo, o HTML vai exibir <body class="best_galleries pictures" id="selection"> — ou seja, esta página pertence à categoria best_galleries (melhores galerias de imagens) e sua identidade é selection. Lembre que o “pictures” que aparece no body é a última palavra da classe, logo reflete o módulo;
  • O segundo par de parênteses ((04_pictures_in_cart)) representa a situação em que o arquivo se encontra. Certas páginas têm comportamento regulado por JavaScript — a de seleção de imagens é uma delas —; logo, apresentam vários estágios de comportamento; para transformar as páginas em HTML estático, tivemos de “congelar” estes estágios de comportamento para que você, designer, possa estilizar com toda precisão possível. Neste caso, esta página-exemplo está no 4º estágio de seu comportamento, quando algumas imagens já foram passadas para o “shopping cart” (o nome pictures_in_cart não aparece no HTML — foi uma escolha arbitrária baseada nessa aparência).

Isso significa que este pacote contém todas as páginas, de todos os módulos, em todas as situações possíveis de comportamento. Você está tendo acesso a tudo o que há no Be on the Net até agora. São, ao todo, 5 módulos distintos, 15 páginas diferentes e 26 situações possíveis.

Convenções:

Certas partes do seu design vão ser integradas diretamente ao HTML: logotipos, chamadas de CSS, declarações de mídia, favicons etc. Mas como o designer não vai ter acesso direto à estrutura (HTML) dinâmica do Be on the Net, a solução é recorrer a certas convenções. Se o designer criar seus arquivos seguindo estas convenções, o sistema do Be on the Net cuida de automatizar tudo e injeta o que for necessário direto no HTML. São elas:

Logotipos

Os logotipos são imagens especiais. Das imagens que o designer pode inserir livremente, os logotipos são as únicas que não se tornam background-images, mas são integradas ao próprio HTML, por causa de sua importância para o Google. Há dois tipos de logotipos: para a visualização em tela e para visualização em impressos.

Para logotipos de tela: o nome do seu arquivo deve ser somente “logo”. O sistema reconhece as extensões .gif, .jpeg, .jpg e .png; logo, vai aceitar seu logotipo se ele se chamar logo.gif, logo.jpeg, logo.jpg ou logo.png. Por exemplo, se você criou um logotipo para tela chamado logo.png, o sistema vai identificá-lo e gerar o seguinte código HTML:

<h1 id="logo_screen">
  <a href="/">
    <img alt="" src="images/logo.png" title="" />
  </a>
</h1>

Nota: as propriedades alt e title vão variar dependendo da configuração da conta do cliente.

Para logotipos de impressão: a convenção é nomear o arquivo como “logo_print”. O sistema reconhece as mesmas extensões do logo de tela, logo vai aceitar seu logotipo se ele se chamar logo_print.gif, logo_print.jpeg, logo_print.jpg ou logo_print.png. Inserir um logotipo de impressão não é obrigatório. Mas fica bonitão!

Se o sistema não encontrar nenhum arquivo de logotipo, vai em vez disso usar o nome da conta do usuário como “logo textual”. Por exemplo, experimente retirar os arquivos de logotipo que vieram neste pacote, e você vai ver que o site vai se chamar somente “Patricia Figueira”.

Nota: agora que você conhece a convenção de logotipos, experimente trabalhar com logotipo textual — retire o logo da Patricia Figueira e use o atributo alt que restou da imagem como um logo textual. E estilize-o como bem quiser.
